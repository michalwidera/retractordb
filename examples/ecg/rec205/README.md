## Files

### `205.hea` — header (plain text, MIT-BIH header format)

Describes the recording:
- `205 2 360 650000` — record name, 2 channels, sampling rate **360 Hz**, 650 000 samples
- Two signals (both stored in `205.dat`):
  - **MLII** (Modified Limb Lead II) — gain 200, 11-bit, baseline 1024
  - **V1** (precordial lead) — gain 200, 11-bit, baseline 1024
- Patient: male, age 59
- Medications: Digoxin, Quinaglute

### `205.dat` — signal data (binary, MIT-BIH format 212)

Binary file, **1 950 000 bytes**, containing interleaved samples from both channels.

Format 212: 12-bit samples packed two per three bytes:
```
[B0][B1][B2] → sample_1 = B0 | (B1 & 0x0F) << 8
               sample_2 = B2 | (B1 >> 4)   << 8
```

650 000 samples × 2 channels = 1 300 000 samples → 1 950 000 bytes. Recording duration: ~30 minutes.

### `205.atr` — annotations (binary, MIT-BIH annotation format)

Binary file, **5 436 bytes**, containing beat labels. Each annotation is a 2-byte little-endian word:
- bits 15–10: beat type (N=normal, V=PVC, A=APC, F=fusion, …)
- bits 9–0: sample difference from the previous annotation

Total: 2656 beats — 2571 normal, 71 PVC, 11 fusion PVC, 3 APC.

### `205.xws` — WAVE viewer script (plain text)

Three-line configuration file for PhysioNet **WAVE** (Waveform Analysis, Visualization, and Editing):
```
-r 205   # record name
-a atr   # annotation file
-p <url> # web preview URL template (SSI)
```

Does not contain signal data — used to launch the `wave` application from PhysioNet.

## Generated files (retractordb)

Source files (`205.hea`, `205.dat`, `205.atr`, `205.xws`) are the original MIT-BIH files —
do not modify them. All retractordb files are generated and share the prefix `rec205`:

| File | Generated by | Description |
|------|-------------|-------------|
| `rec205` | `mitbih2rdb.py` | binary data (int32 LE) |
| `rec205.desc` | `build.sh` | descriptor |
| `rec205-replay.rql` | `mitbih2rdb.py` | RQL playback script |

Generate all three by running from the project root:

```bash
bash examples/ecg/build.sh
```

The `rec` prefix (instead of `205`) is required because the DESC grammar token `FILENAME`
cannot be distinguished from `DECIMAL` when a name consists only of digits — ANTLR4 matches
`205` as `DECIMAL` first, causing a parse error in `xtrdb` and `xretractor`.

### `rec205` — binary data (retractordb, int32 little-endian)

Produced by `mitbih2rdb.py` from `205.dat`.  
650 000 records × 2 fields × 4 bytes = **5 200 000 bytes**.  
Each record holds one sample pair: MLII and V1 as signed 32-bit integers (raw ADC values;
physical value in mV = (ADC − baseline) / gain = (ADC − 1024) / 200).

Format 212 decode formula applied during conversion:
```
sample_MLII = B0 | ((B1 & 0x0F) << 8)   sign-extended from 12 bits
sample_V1   = B2 | ((B1 >>  4) << 8)    sign-extended from 12 bits
```

### `rec205.desc` — descriptor (TYPE DEVICE)

Produced by `build.sh` to match the file that `xretractor` auto-creates on first run:

```
{	INTEGER MLII
	INTEGER V1
	REF "rec205"
	TYPE DEVICE
}
```

`TYPE DEVICE` opens the data file through `binaryDeviceRO`, which reads sequentially and wraps
back to the start after the last record (loop replay). `REF "rec205"` is the binary filename
from the `DECLARE FILE 'rec205'` clause — xretractor stores it in the descriptor automatically.

### `rec205-replay.rql` — RQL playback script

Produced by `mitbih2rdb.py`.

```rql
DECLARE MLII INTEGER, V1 INTEGER STREAM ecg, 1/360 FILE 'rec205'

SELECT ecg.MLII, ecg.V1 STREAM s205out FROM ecg VOLATILE
```

- `STREAM ecg, 1/360` — sampling interval 1/360 s (= 360 Hz), matching the original recording
- `FILE 'rec205'` — references `rec205.desc` and the `rec205` binary data file
- `VOLATILE` — output stream is not written to disk

Run from the `rec205/` directory so that relative paths resolve correctly:

```bash
cd examples/ecg/rec205
xretractor rec205-replay.rql -m 1000   # replay 1000 samples (~2.8 s of ECG)
```
